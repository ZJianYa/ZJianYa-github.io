和所有软件一样， MySQL 也是模块化的，服务端大致划分如下：  
* 服务层  
  * 连接器  
    * 8 小时问题，在大查询或者长时间运行之后重新连接的必要性  MySQL_rest_connection  
    * 不建议查询缓存区，但不是绝对不能用缓存    
      `mysql> select SQL_CACHE * from T where ID=10；`
  * 分析器  
    * 完成的是编译器的"前端"功能  
    * 也可能有权限校验  
  * 优化器  
    * 比如 join 的顺序  
  * 执行器  
    * 会有权限校验  
    * row_examined 表示扫描了多少行（引擎返回的数据），和引擎扫描了多少行是不一样的
* 引擎层
